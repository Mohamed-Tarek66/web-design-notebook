As shown bellow there is 2 locations of the files local & remote repo. The local location is the files on the pc related to the remote repo. The local location itself consists of 3 areas working directory, staging area & local repo.

The git status command is used to determine the files different from the files in the remote repo and at wich location of the mensioned above they are located.

The git init command is used to declair the current file as a working directory and make a local repo out of it. While the git clone the location of the remote repo in github command is used to declair the current file as a working directory by linking it to that remote repo and download all the files in that remote repo into this file.

The git add command is used to copy the files in the working directory that is different from files in the local repo to the staging area. The following table showes its forms:

form indication
git add fileLocation that file will be added to the staging area
git add fileLocation fileLocation fileLocation .... these files will be added to the staging area
git add * all files different from files in the local repo will be added to the staging area

The git commit -m "message" command is used to put all files added to the staging area to the local repo attached with this message.

The git push remoteRepoName branchName command is used to put all commited files in that branch of that remote repo. The commited message will also be attached to that push at the remote repo.

The git pull remoteRepoName command is used to put all files in remote repo that is different from the files in the local repo into te working directory.

The git push -u remoteRepoName branchName command is used to pull the current files in the remote repo then push the files in your local repo.

The git restore --staged command is used to remove certain files from the staging area before in order not to commit it to the local repo.

form indication
git restore --staged fileLocation that file will be removed from the staging area
git restore --staged fileLocation fileLocation fileLocation .... these files will be removed from the staging area
git restore --staged * all files currently in the staging area will be removed

The git clean -n command is used to determine the files different from the files in the remote repo located in the workong directory.

The git clean -f command is used to ignore the files different from the files in the remote repo located in the workong directory, ie consider it is not part of the working directory.
