The video tag is used to insert a video file. It is written in the following form:

<video>
  <source src="the location of the video file" type="the mime type of the video file">
  <source src="the location of the video file" type="the mime type of the video file">
  <source src="the location of the video file" type="the mime type of the video file">
  <source src="the location of the video file" type="the mime type of the video file">
  ..............
  <track src="the location of the track file" kind="the type of the track" srclang="the language of the track" label="the title of the track">
  <track src="the location of the track file" kind="the type of the track" srclang="the language of the track" label="the title of the track">
  <track src="the location of the track file" kind="the type of the track" srclang="the language of the track" label="the title of the track">
  <track src="the location of the track file" kind="the type of the track" srclang="the language of the track" label="the title of the track">
  ..............
</video>

Every source tag provides a video file. In order to make sure that the web browsers provides the video file with no problems we should put several source tags with different mime types. The src attribute determines the location of the video file: its value can be a link or a location on the pc. The type attribute determines the mime type of the video file which can be detemined through this link.

Every track tag provides a track file which are text that shows with the video. The attributes used in conjuction with the track tag are:

attribute usage
src determines the location of the track file: its value can be a link or a location on the pc
label determines the title of the track
srclang determines the language of the track file
kind determines the type of the track and it takes one of the following values:
  • "subtitle"
  • "caption" which is for people with hearing prolems
  • "chapters" which determines the start of every part of the video
  • "description" which provides the description of the video
  • "metadata" which determines the data of the video

Common attributes used in conjuction with the video tag are:

attribute usage
controls used to show the controls of the video
autoplay indicates that the video will autoplay after the loading of the website
loop used to make the video automatically replay after it is finished
muted indicates that the video will be muted by default
poster determines the location of the picture that will apper at the video screan before the user plays the video
preload indicates when the video file loads and it takes one of 3 values:
  • "auto" which indicates that the file will load with the website
  • "none" which indicates that the file will load only when played
  • "metadata" which indicates that the file will load only when played but its data will load with the website
