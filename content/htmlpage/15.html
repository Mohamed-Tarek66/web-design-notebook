The input tag is a self-closed tag used to recieve data from the user in several types. The type attribute determines the type of the input such as "text", "button", "checkbox" ....ect. It is written in the following form: <input type="the type of the input"/>

The types of the input tag are summarized in the following table:

type usage showcase
text used as an input textbox for the user to input data as text
password used as an input textbox for the user to input a password with its litters shown as stars
number used as an input textbox for the user to input data as text but only accepts numbers
email used as an input textbox for the user to input data as text but only accepts text in form of Email adress
url used as an input textbox for the user to input data as text but only accepts text in form of URL link
tel used as an input textbox for the user to input data as text but indicates that its a telephone number
time used as an input textbox for the user to input data as text in form of time "hr:min:sec"
date used as an input textbox for the user to input data as text in form of date "mon/day/year"
month used as an input textbox for the user to input data as text in form of date "mon/year"
datetime used as an input textbox for the user to input data as text in form of date "mon/day/year" then "time hr:min:sec"
datetime-local used as an input textbox for the user to input data as text in form of the pc local date & time
week used as an input textbox for the user to input data as text in form of date "week, year"
search used as an input textbox for the user to input data as text but it has an x icon to remove its content
checkbox every checkbox shows a square that the user can check to send data in form of name = value for every checked checkbox
radio every radio shows a sphere that the user can check to send data in form of name = value for the checked radio as the user can only check 1 of different radios if they have the same name
range it is a slide bar to input a value within a renge
color it allows the user to input a certain color as its value
hidden it is a hidden input the user cant interact with and it gets its value from the user's browser or pc
button on clicking on it it will perform a programming function
submit on clicking on it it will send the values of all inputs in its form to the programing file in form of "name = value"
image same as submit but it is a picture in steed of a button
reset on clicking on it it will reset the value of all inputs in its form to their specified values
file allows the user to upload a file as an input

Attributes used in conjuction with the input tag with its different types are:

attribute usage
name distanguishes it from other inputs and the value of the input is sent to the programming file in this form "name = value"
value represents the value of the input sent to the programming file in the form "name = value" also represents the containt text of textbox-alike and button-alike inputs
required the user must put a value for this input
disabled the user can not put a value for this input and its value will not be submitted to the programming file
autofocus when the website loads this input will be focused upon
form used to make this input treated as a part of the form that its name is the value of this attribute even if the input is outside the form

Common attributes used in conjuction with the textbox-alike input tag are:

attribute usage
maxlength determines the max number of litters that the the user is allowed to input
placeholder gray text written in the textbox and will automatically be removed when the user inters a value
readonly the user can not change the value of the textbox but its value will still be submitted to the programming file
autocomplete it accepts one of 2 values:
  • "on" which indicates that the prevous values inputed by the user will appear to him while inputing the value of the textbox
  • "off" which indicates that the prevous values inputed by the user will not appear to him while inputing the value of the textbox

Common attributes used in conjuction with the number & range input tag are:

attribute usage
min indicates the minimum value that can be inputed by the user and the minimum value of the range
max indicates the maximum value that can be inputed by the user and the maximum value of the range
step indicates the difference between every range choosable number and the next choosable one and the amount it will add to or remove from the current number if u click the up and down button in the number input tag

A common attribute used in conjuction with the url & email input tags is novalidate that removes the restriction of the value of the input.

A common attribute used in conjuction with the file & email input tags is multiple that allows the user to submit multiple files or input multiple emails separated by ", ".

A common attribute used in conjuction with the text, date, search, url, tel, password & email input tags is pattern which accepts a regular expression as a value to restrict the user to input a value according to the regular expression.

Common attributes used in conjuction with the submit input tag are:

attribute usage
formaction determines the location of the programming file that will recieve the input values of this submit
formmethod determines the way the programming file that will recieve the input values of this submit it accepts 2 values:
  • "post" which will send the data to the programming file and post it besides the widsite address
  • "get" which will only send the data to the programming file
formnovalidate removes the restriction of the value of the inputs that will be submitted by this submit.
formtarget It holds 2 values:
  • "_self" which makes the values of the inputes is submitted in the same web tab
  • "_blank" which makes the values of the inputes is submitted in a new web tab
